## Entity Overview

| Entity           | Role                            | Key relationships                   | Status machine                                             |
| ---------------- | ------------------------------- | ----------------------------------- | ---------------------------------------------------------- |
| User             | System actor (Staff / Client)   | has many Bookings, Carts, Favorites | —                                                          |
| Service          | Offerings the business provides | has many Bookings, Carts            | —                                                          |
| Booking          | Core transaction                | belongs to User, Service            | PENDING → CONFIRMED → IN_PROGRESS → COMPLETED or CANCELLED |
| Cart             | Staged service items            | belongs to User, Service            | —                                                          |
| DiscountCode     | Cart discount configurations    | has many Usages                     | ACTIVE (Boolean toggle)                                    |
| PointsRedemption | Loyalty points conversion       | belongs to User                     | PENDING → APPLIED → EXPIRED                                |
| ReferralCode     | User recruitment pipeline       | belongs to User                     | —                                                          |
| Rating           | User reviews on Services        | belongs to User, Service            | PENDING → APPROVED → REJECTED                              |

## Key Business Rules (enforced at data layer)

1. **Monetary Precision:** Base currency limits calculation drifts. USD Cents (`Int`) natively enforced.
2. **Referral Reward Logic:** Only verified application rules convert referral events to user point credits.
3. **Soft-Deletes:** Administrative User and Service deletions populate `deletedAt`, retaining transactional ties historical logging intact.
4. **Time & Dates:** Always assume API payload translation to UTC values. Database querying defaults to `startsAt` filtering composites.

## Monetary Fields

- `Service.price` (Int) - Cents (USD)
- `DiscountUsage.discountAmount` (Int) - Cents (USD)
- `DiscountUsage.cartTotal` (Int) - Cents (USD)
- `DiscountUsage.finalTotal` (Int) - Cents (USD)
- `DiscountCode.value` (Int) - Mixed; If `DiscountType` is `FIXED`, stored in Cents (USD); if `PERCENT`, basis point equivalent.
- `PointsRedemption.discountAmount` (Int) - Cents (USD)

## Status Vocabularies

**BookingStatus**

- `PENDING`: Initial state upon creation.
- `CONFIRMED`: Acknowledged via Staff or Auto-policy logic.
- `IN_PROGRESS`: Marked when service actively begins.
- `COMPLETED`: Terminal. Service fulfilled.
- `CANCELLED`: Terminal. Cancelled by Client or Staff (`cancelledBy` trackable).

**RatingStatus**

- `PENDING`: Content moderation cue.
- `APPROVED`: Display publicly.
- `REJECTED`: Hidden from public.

**UserRole**

- `SUPER_ADMIN`, `ADMIN`, `STAFF`, `CLIENT`

**DiscountType**

- `PERCENT`, `FIXED`
